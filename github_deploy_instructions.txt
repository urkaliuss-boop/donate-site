ИНСТРУКЦИЯ ПО ПУБЛИКАЦИИ САЙТА

Эта инструкция описывает, как сохранить ваш код на GitHub и бесплатно выложить сайт в интернет.

============================================================
ЧАСТЬ 1. КАК ЗАЛИТЬ КОД НА GITHUB
============================================================

Вам понадобятся:
1. Установленный Git на компьютере (скачайте с https://git-scm.com/downloads и установите, нажимая "Next").
2. Аккаунт на GitHub (https://github.com/).

Шаги:

1. Перезапустите ваш редактор кода (VS Code) после установки Git, если вы его только что поставили.
2. Перейдите на сайт GitHub (https://github.com) и войдите в свой аккаунт.
3. В правом верхнем углу нажмите "+" -> "New repository".
   - Укажите имя репозитория (например, `scp-donates-site`)
   - Выберите "Private" (закрытый код) или "Public" (открытый) на ваше усмотрение.
   - НЕ ставьте галочку "Add a README file" (нажмите просто Create repository).
4. Откройте терминал в папке вашего проекта (D:\scp site) и выполните эти команды по очереди:

   git init
   git add .
   git commit -m "Первый коммит проекта"
   git branch -M main
   git remote add origin ССЫЛКА_НА_ВАШ_РЕПОЗИТОРИЙ_С_GITHUB_ИЗ_ШАГА_3
   git push -u origin main

(Внимание: если Git запросит авторизацию, войдите через браузер).
Теперь ваш исходный код сохранен в облачном хранилище GitHub.


============================================================
ЧАСТЬ 2. ГДЕ И КАК ЛУЧШЕ ХОСТИТЬ (GITHUB PAGES ИЛИ VERCEL)
============================================================

ВАЖНО: Ваш сайт написан на Next.js (современный мощный фреймворк) и использует подключение к Supabase (аутентификация).
Размещать его на GitHub Pages можно, но это сильно "обрежет" функциональность (потребует создать статичную версию `output: "export"`), так как GitHub Pages умеет отдавать только простые HTML файлы, а вашему сайту нужен Node-сервер.

ЛУЧШИЙ ВАРИАНТ (Рекомендуется) — Бесплатный хостинг от Vercel:

1. Перейдите на https://vercel.com и зарегистрируйтесь (кнопка "Continue with GitHub").
2. Нажмите "Add New..." -> "Project".
3. Vercel покажет список ваших репозиториев с GitHub. Найдите ваш `scp-donates-site` и нажмите "Import".
4. Самое важное: Откройте блок "Environment Variables" и добавьте ключи от Supabase, без них сайт будет белым экраном:
   - Имя: NEXT_PUBLIC_SUPABASE_URL, Значение: (ваш URL из .env.local)
   - Имя: NEXT_PUBLIC_SUPABASE_ANON_KEY, Значение: (ваш длинный ключ из .env.local)
5. Нажмите "Deploy". 
Vercel сам всё настроит, соберет сайт и выдаст ссылку (вида scp-donates-site.vercel.app), которую потом можно заменить на свой красивый домен.


ЕСЛИ ВСЁ ЖЕ ХОТИТЕ GITHUB PAGES (Не рекомендуется для этого сайта):

1. Откройте файл `next.config.js` и добавьте строку `output: "export",`. Должно получиться так:
   const nextConfig = {
     output: "export",
     // остальное
   }
2. В файле `package.json` в разделе scripts измените:
   "build": "next build"

3. На GitHub в настройках вашего репозитория перейдите в "Settings" -> "Pages".
4. В разделе "Source" выберите "GitHub Actions".
5. GitHub сам предложит "Next.js" workflow. Нажмите "Configure" и закоммитьте предложенный файл.
6. Вы должны прописать ваши Supabase ключи в GitHub Secrets (Settings -> Secrets and variables -> Actions), иначе билд упадет.
   Назовите их NEXT_PUBLIC_SUPABASE_URL и NEXT_PUBLIC_SUPABASE_ANON_KEY.
7. Сайт появится по ссылке `https://ВАШ_НИК.github.io/scp-donates-site/`. 
Из-за статичности многие функции вроде работы с сервером платежей не смогут запускаться.
